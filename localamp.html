Some Answers to Questions:
You may have some questions, there might be some terminology you don't understand, and this section might have your answers

  • What does *AMP stand for?
    AMP stands for Apache (the actual server software), MySQL (database software that's used by many sites and web based programs) and PHP (a programming language for the web). The * is used because we don't know what operating system you're running, but the options are L (Linux), M (Mac OS X), or W (Windows).
  • What is a stack?
    A stack is a way of describing all of the software together, generally people have a LAMP stack, which is all of the software described above, just with a fancy word after it.
  • What is different about a LAMP a MAMP or a WAMP stack?
    Nothing, other than the operating system they're running on, you can run the same software on each stack. Some configuations may be different (but those are set by both the user and the software being run to host the stack).
  • Why would I want a *AMP stack?
    An *AMP server is useful for local development of php/mysql or higher level web development projects. I essentially gives you a server to play with that’s only accessible on your computer, so it’s just for you.
  • How do I know if it works?
    Open a web browser and head over to 127.0.0.1, if you get any sort of It Works page, you’re golden (though some may have different pages shown)

LAMP - Linux (Debian Baesd)
The distribution used by OPRF Tech is Ubuntu, which is based on Debian, and has apt-get. Other distributions may use a different program to fetch new applications (like yum or pacman), in which case, try substituting apt-get with one of those.

  • Open your terminal and type the following:
    sudo apt-get install apache2 php5 php5-gd mysql-server mysql-client php5-mysql libapache2-mod-php5 php5-curl php5-xmlrpc php-pear
  • Bravo! You've now got a LAMP server installed, your www directory (the one that holds all of the files for your local server) is located at /var/www. You may want to run another command to change the premissions to let you modify files inside of it
    sudo chmod -R 0777 /var/www

MAMP - Mac OS X
You need to be running Tiger (10.4.0) or higher to use MAMP.
MAMP does not load at startup/login

  • You'll need to first grab the MAMP app, over at http://mamp.info, and download just the standard MAMP (you don't need MAMP Pro, unless you want to pay for it).
  • Drag the MAMP.app file out of the DMG and place it in your applications directory.
  • Now you need to tweak a few settings, so open up MAMP and head into the preferenecs.

    • Under the Ports tab, change the Apache port to 80. 80 is the default port that web browsers look for when visiting a website.
    • Under the Apache tab, change the Document Root to the directory that you'll be using to keep the files for your local server. I suggest using /users/[your name]/www.
  • Awesome! You've got the server setup, now just open up MAMP and start Apache and MySQL and you're good to go.

WAMP - Windows
Windows has loads of options, however the easiest (and the one that we're going to ues) is called EasyPHP. If you want to, you can install EasyPHP on a flash drive and take it with you.
EasyPHP does not start at startup/login

  • You're going to need to nab a copy of EasyPHP. You can get the newest copy through thier website, http://www.easyphp.org, but it doesn't work as well as some of the legacy versions. The version I use and suggest you use is 5.3.2i, which can be downloaded from the archives.
  • Run the installer, pick whatever directory you want to install it in (this is both where you're going to keep your files and where the server files are located), I suggest using C:\Users\[your name]\localhost.
